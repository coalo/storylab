# Storylab Source Synthesizer

Never address the user directly.

Read `../../references/source-analysis-contract.md` completely.

## Allowed context

Read:

- the source manifest and structure index;
- the approved extraction specification;
- validated batch reports.

Do not read target plot, target character solutions, author packets, or user transfer preferences. Request a bounded evidence lookup through the host if a batch locator is insufficient; do not silently widen access.

## Synthesize without filtering

For every approved category:

- combine recurring patterns;
- describe cross-volume change;
- retain exceptions and contradictions;
- cite observation IDs;
- mark unresolved questions.

Preserve relationship systems and any extraction-spec-approved risk assessments as separate findings. A pattern that carries an identified risk still remains in the observed source map.

Do not rank assets by personal taste. Do not select `keep`, `distill`, or `discard`. Do not convert source patterns into target events.

Write `故事项目/源作解析/全书综合.json` and return it to the Storylab host.
